# Step-by-Step Guide: Filling Edge Impulse Studio Dashboard

This guide provides step-by-step instructions for filling in the project information in Edge Impulse Studio dashboard.

## Step 1: Access Project Info

1. Log in to [Edge Impulse Studio](https://studio.edgeimpulse.com)
2. Navigate to your project: **Diabetic Retinopathy Detection**
3. In the left sidebar, click on **"Project info"** (should be at the top, already selected)

## Step 2: Fill "About this project" Section

1. Scroll down to the **"About this project"** section
2. You'll see a text editor with Markdown support
3. Copy the content from `EDGE_IMPULSE_DASHBOARD_README.md` file
4. Paste it into the text editor

**Note**: The text editor supports Markdown syntax. You can use:
- `#` for headings
- `**text**` for bold
- `-` for bullet points
- `|` for tables
- `✅` for checkmarks

## Step 3: Verify Markdown Formatting

After pasting, verify that:
- Headings are properly formatted (using `#`, `##`, `###`)
- Bold text appears correctly (`**text**`)
- Bullet points are displayed as lists
- Tables (if any) are formatted correctly
- Emojis (✅, ⬆️) display properly

## Step 4: Save Changes

1. The changes should auto-save, but verify by:
   - Clicking outside the text editor
   - Or refreshing the page to confirm content persists

## Step 5: Set Project Visibility (Optional)

1. In the **"Sharing"** section (right column)
2. Ensure **"Public"** is selected (for hackathon submission)
3. Copy the public link if needed: `https://studio.edgeimpulse.com/public/[YOUR_PROJECT_ID]/live`

## Step 6: Add Project Tags (Optional)

1. Click the **"+ New tag"** button near the project title
2. Add relevant tags like:
   - `healthcare`
   - `diabetic-retinopathy`
   - `edge-ai`
   - `image-classification`
   - `medical-imaging`

## Step 7: Verify Public Access

1. Open the public link in an incognito/private browser window
2. Verify that the "About this project" section displays correctly
3. Ensure all formatting is preserved

## Markdown Syntax Reference

For reference, here are common Markdown elements you can use:

### Headings
```markdown
# Heading 1
## Heading 2
### Heading 3
```

### Bold and Italic
```markdown
**bold text**
*italic text*
```

### Lists
```markdown
- Bullet point
- Another point

1. Numbered item
2. Another item
```

### Code
```markdown
`inline code`
```

### Links
```markdown
[Link text](https://example.com)
```

### Tables
```markdown
| Column 1 | Column 2 |
|----------|----------|
| Data 1   | Data 2   |
```

### Checkmarks
```markdown
✅ Completed
- [x] Task done
```

## Troubleshooting

**Issue**: Markdown not rendering correctly
- **Solution**: Ensure proper spacing between elements (blank lines between sections)

**Issue**: Tables not displaying
- **Solution**: Ensure proper table syntax with pipes (`|`) and dashes (`---`)

**Issue**: Content not saving
- **Solution**: Check internet connection, try refreshing page, or copy content before refreshing

**Issue**: Public link not working
- **Solution**: Verify project is set to "Public" in Sharing settings
